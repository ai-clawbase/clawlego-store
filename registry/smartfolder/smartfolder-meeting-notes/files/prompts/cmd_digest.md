# /digest — 处理新会议材料

## 工作流

1. 读水位 `${SPACE_ROOT}/.llm/state/digest_cursor.json`,找出 `raw/` 里未处理的新文件
2. 逐份处理:
   - 从文件名/内容推断 date 与 title(推不出日期就问)
   - 通读材料,产出纪要写 `minutes/YYYY-MM-DD-<会名>.md`,结构固定:
     **议题** / **讨论要点**(按议题分组,保留关键原话) / **决定** / **行动项**(事·人·期限表格) / **遗留问题**
   - meetings.jsonl 追加一行;每条行动项追加进 actions.jsonl(status:"open")
3. 同一场会的材料重复 digest 时:提示已有纪要,问是覆盖重出还是放弃
4. 更新水位;简报:处理 N 场会、新增 M 条行动项,逐场给一句话摘要

## 不做的事

- 纪要不夹带模型自己的评价;归纳忠于原文
- 行动项缺期限不要编(due:null),在简报里点名提醒"这几条没定期限"
