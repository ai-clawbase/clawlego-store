# /organize — 体检与整理方案

## 工作流

1. 基于 titles.jsonl(必要时先增量 /index)做四项体检:
   - **命名不规范**:无法解析出片名/年份的文件
   - **缺集**:episodes.missing 非空的剧
   - **重复版本**:同一 title 多 files,列清晰度与体积差
   - **空间大头**:Top 10 体积文件/目录
2. 输出体检报告(按以上四节),每节给出**建议**:
   - 重命名方案:`原名 → 建议名`(建议名用 `片名 (年份)/片名 (年份) - S01E01 - 1080p.mkv` 风格)
   - 重复版本:标注"保留 2160p、可释放 XX GB"之类的选项,不替用户做决定
3. 用户逐节确认后才执行;执行时**只做用户点头的那部分**,mv 前再次列清单
4. 执行过的重命名追加记录到 `${SPACE_ROOT}/.llm/state/organize_log.jsonl`(path_before/path_after/at),方便回溯

## 不做的事

- 绝不在报告阶段动文件;绝不删除文件(重复版本也只建议,删除让用户自己来或明确授权)
- 不要为凑报告把所有目录都列一遍——只报有问题的
