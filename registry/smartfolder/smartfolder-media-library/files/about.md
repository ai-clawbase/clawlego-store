把 NAS 上的电影/剧集目录"提升"为影视库后,每个视频都会被识别:片名、年份、季/集、分辨率、字幕情况。乱糟糟的下载命名也能问得出来。

# 你能用它做什么

- "我有没有存过《沙丘2》?是什么清晰度的?"
- "哪些剧只下了一半(缺集)?"
- "去年存的电影里评分类型是科幻的有哪些?"
- "哪些文件命名不规范,帮我出个重命名方案"

# 落盘结构(agent 维护)

```
<影视库根>/
├── Movies/沙丘2.Dune.Part.Two.2024.2160p.mkv   ← 视频文件,agent 只读
├── TV/怪奇物语/S04/...
└── .llm/
    ├── manifest.yaml
    ├── about.md
    ├── index/titles.jsonl     ← 每部作品一行(git-ignored,可重建)
    └── state/index_cursor.json
```

# 命令

- `/index` — 重建或增量建库:从文件名/目录结构识别作品信息
- `/search <query>` — 按片名/年份/类型/清晰度检索
- `/organize` — 体检报告:命名不规范、缺集、重复版本、占用大头;给出整理方案(只建议,确认后才动)

# 注意

- agent 不会读取视频内容,只看文件名/目录/伴随文件(nfo/字幕/海报)
- 任何重命名/移动/删除都先出方案、等确认
