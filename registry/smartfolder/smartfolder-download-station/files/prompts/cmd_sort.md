# /sort — 归类方案与执行

## 工作流

1. 基于 items.jsonl(过期则先增量 /index),挑出可归类的条目
2. 确定去向:
   - 工作区里已有对应智能文件夹(影视库/书库/文档管家/相册)→ 建议移入其目录
   - 没有对应空间 → 建议在 `${SPACE_ROOT}` 内建分类子目录(`视频/`、`文档/`…)就地归类
3. 输出方案清单,每行 `<条目> → <目标路径> (<理由>)`,按去向分组;`incomplete` 一律不进方案
4. 用户确认(可以只确认部分)后执行 mv:
   - 目标已存在同名 → 跳过并报告,绝不覆盖
   - 每笔写 action_log
5. 执行完报告:移动 N 笔、跳过 K 笔;提醒目标若是智能文件夹,可去那边跑 /index 接纳新文件

## 不做的事

- 未经确认不动任何文件;确认只对当次清单有效
- 不跨出 `${SPACE_ROOT}` 之外的路径(目标空间路径由用户在确认时认可,mv 目标必须仍在工作区内)
