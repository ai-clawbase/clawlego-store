把下载目录"提升"为下载整理站后,agent 会认得每个下载物是什么(影视/软件/文档/压缩包),该归到哪,哪些是垃圾。**方案先行、确认后动**——它不会擅自移动你的文件。

# 你能用它做什么

- "下载目录里都堆了什么?多久没动过的有哪些?"
- "把能归类的都归一归"(出方案 → 你确认 → 执行)
- "哪些是装完就没用的安装包/解压完的压缩包?"
- "这目录还剩多少能清出来的空间?"

# 落盘结构(agent 维护)

```
<下载根>/
├── (各种下载物,agent 默认只读)
└── .llm/
    ├── manifest.yaml
    ├── about.md
    ├── index/items.jsonl       ← 盘点清单(git-ignored,可重建)
    └── state/
        ├── index_cursor.json
        └── action_log.jsonl    ← 执行过的移动/删除记录,可回溯
```

# 命令

- `/index` — 盘点:每个条目识别类型、来源、最后访问时间
- `/sort` — 归类方案:哪个该去影视库/书库/文档管家,确认后移动
- `/clean` — 清理方案:重复、残留安装包、解压后的压缩包、超期未动的,确认后删除

# 注意

- 移动/删除永远先出清单等确认;执行记录落 action_log 可回溯
- 跨空间归类(移去影视库等)需要目标空间也在工作区内,agent 会给出目标路径让你点头
