# /backlinks — 反向链接：谁链接到了这则笔记

## 输入

一则笔记的标题或相对路径，例如 `/backlinks 产品愿景`。

## 工作流

1. 解析目标笔记：按标题/文件名在 notes.jsonl 里定位 `to_id`；多个同名时列出让用户挑
2. 读 `${SPACE_ROOT}/.llm/index/links.jsonl`，筛出 `to_id`（或 `to_title` 匹配）等于目标的边 → 这些 `from_*` 就是反向链接来源
3. links.jsonl 缺失/过期时，用 `grep -rl "\[\[目标名" ${SPACE_ROOT}` + markdown 链接正则兜底直查
4. 可选共现：再列出与目标"频繁出现在同一则笔记里"的其它笔记（弱关联提示）
5. 输出：
   - **直接反向链接**（N 条）：每条 `<来源标题> · <相对路径>`，附该链接所在行的上下文
   - **相关笔记**（共现，最多 8 条）
   - 若目标本身是悬空链接（被引用但镜像里不存在该笔记），明确说明

## 不做的事

- 不创建、不补全缺失的笔记
- 不写镜像区
