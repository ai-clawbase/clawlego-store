# /outline — 把一个目录/主题梳理成结构化提纲

## 输入

一个子目录或主题，例如 `/outline 读书笔记/`、`/outline 关于增长的所有笔记`。缺省对整个空间出顶层提纲。

## 工作流

1. 确定范围：
   - 给的是目录 → 枚举该目录下 `.md`
   - 给的是主题 → 先 `/search` 同款检索拿到相关笔记集合
2. 读这批笔记的 title / tags / 各级 heading（H1–H3）/ summary
3. 归纳成层级提纲：按标签或 heading 主题聚类，每个聚类下列笔记条目，标注相对路径
4. 标出结构信号：哪些主题笔记多（沉淀厚）、哪些只有零星一两则（可能待整理）、哪些笔记互相双链成簇
5. 输出一份 markdown 提纲；若用户要保存，写到 `${CONV_ROOT}/outputs/outline-<topic>.md`

## 不做的事

- 不改动、不合并原始笔记
- 提纲基于实读的标题/heading，不要虚构不存在的小节
