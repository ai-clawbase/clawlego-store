# /search — 按概念或符号检索代码库

用法：`/search <自然语言问题或符号名>`，例如 `/search 路由注册在哪` 或 `/search AtomicWriteFile`。

## 工作流

1. 先判断查询类型：
   - **符号/标识符**（看起来像函数/类型名）→ 在 `${SPACE_ROOT}/.llm/index/symbols.jsonl` 里按 `name` 精确 + 模糊匹配。
   - **概念/自然语言** → 在 `${SPACE_ROOT}/.llm/index/files.jsonl` 的 `summary`/`exports` 里做语义匹配，挑出最相关的若干文件。
2. 如果索引不存在或明显过期（cursor 时间远早于源码 mtime），**先提示用户跑 `/map`**，同时用 `rg` 实扫兜底回答本次问题。
3. 对命中的符号/文件，用 `rg -n` 在源码里定位精确行，给出 `file:line` 可点击引用。
4. 输出一份排序后的结果列表（最相关在前）：每条给 `file:line`、一句话为什么相关、必要时贴 3–8 行关键代码片段。

## 规则

- 引用一律用 `file:line` 形式，路径相对 `${SPACE_ROOT}`
- 不要编造索引里没有、源码里也没扫到的符号；找不到就如实说"未命中，建议 /map 刷新或换关键词"
- 检索是只读操作，绝不改动任何文件
