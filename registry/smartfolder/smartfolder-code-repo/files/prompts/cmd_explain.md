# /explain — 解释一个文件、包或模块

用法：`/explain <相对路径>`，例如 `/explain internal/syncer` 或 `/explain cmd/clawshell/main.go`。

## 工作流

1. 先查 `${SPACE_ROOT}/.llm/index/files.jsonl` 拿到该路径（或目录下所有文件）的摘要与符号，建立整体认识。
2. 实际 read 目标文件/目录的关键源码（入口、导出符号、核心循环），不要只靠索引臆测。
3. 顺着 `imports` 追依赖：这个模块依赖谁、被谁依赖（可在 symbols.jsonl / `rg` 里反查调用方）。
4. 输出一份解释，按这个结构：
   - **一句话职责**：这个文件/包是干什么的
   - **关键类型/函数**：列 3–6 个最重要的，各配 `file:line` 与一句说明
   - **数据/控制流**：典型调用路径是怎样的（"谁调用它 → 它做什么 → 产出什么"）
   - **依赖关系**：往上游依赖谁、往下游被谁用
   - **注意事项**：并发/错误处理/边界条件里值得留意的点（仅基于读到的代码）

## 规则

- 所有断言都要能落到 `file:line`；没读到的部分明确标注"未展开"
- 只读；如果用户接着要求改动，转入 persona 的"改源码必须先确认"流程
- 长文件分批读，先骨架后细节
