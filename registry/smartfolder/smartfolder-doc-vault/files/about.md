把合同、票据、证件、说明书等杂乱文档"提升"为档案库后,每份文档都会被建档:类型、对方、金额、关键日期、到期提醒。找文档不再翻文件夹,直接问。

# 你能用它做什么

- "我的护照什么时候到期?"
- "找一下 2025 年和房东签的租房合同"
- "这台洗衣机的保修单还在不在?"
- "下个季度有什么要续费/到期的?"

# 落盘结构(agent 维护)

```
<档案根>/
├── 合同/租房合同-2025.pdf      ← 原始文档,agent 只读
├── 票据/...
├── 证件/...
└── .llm/
    ├── manifest.yaml
    ├── about.md
    ├── index/docs.jsonl       ← 每份文档一行(git-ignored,可重建)
    └── state/index_cursor.json
```

# 命令

- `/index` — 重建或增量建档:读取文档抽取类型/对方/金额/日期
- `/search <query>` — 按类型/对方/时间/金额检索
- `/expiry` — 盘点即将到期的:证件、合同、保修、订阅

# 注意

- 文档可能含敏感信息(身份证号/银行卡号);索引里只存**脱敏摘要**(如尾号),不存完整号码
- agent 不移动/重命名原始文件;归类建议会先征求确认
