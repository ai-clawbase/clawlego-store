# /search — 按类型/对方/时间/金额检索文档

## 输入

自然语言查询:"2025 年的租房合同" / "和顺丰有关的票据" / "金额超过一万的发票" / "护照在哪"

## 工作流

1. 解析查询 → docs.jsonl 字段过滤:doc_type、party/title 模糊、dates 范围、amount 阈值
2. 读 `${SPACE_ROOT}/.llm/index/docs.jsonl` 过滤
3. 索引缺失/过期时提示先 `/index`,同时用文件名兜底搜一次
4. 输出:每条 `<doc_type> · <title> · <对方> · <关键日期> · <相对路径>`,按相关度排,封顶 20 条
5. 用户指明某一份后,可以读原件回答细节问题(完整号码仍须用户明确要求才念)

## 不做的事

- 不要为了回答把所有原件读一遍——先用索引
