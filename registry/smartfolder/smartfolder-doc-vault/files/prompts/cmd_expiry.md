# /expiry — 到期盘点

## 工作流

1. 读 `${SPACE_ROOT}/.llm/index/docs.jsonl`,收集所有带 `dates.expires`(或语义等价的续费/年检/保质日期)的条目
2. 按距今天数分三档输出:
   - **已过期**:红线列出,标注过期多久
   - **90 天内到期**:重点段落,每条带"建议动作"(续签/换证/续费)
   - **更远的**:一行带日期即可
3. 没有任何到期信息时如实说明,提示哪些 doc_type 通常该有到期日(证件/合同/保修)但索引里缺失,建议补跑 /index 或人工补充
4. 用户要的话,把 90 天内的清单写成 `${CONV_ROOT}/outputs/expiry-<日期>.md` 供设置提醒用

## 不做的事

- 不直接创建系统提醒/日程(那是用户和 harness 的事;你只产出清单)
- 日期不明的不要猜——列入"日期待确认"
