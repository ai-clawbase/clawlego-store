# /write — 引导式写日记

## 工作流

1. 确定日期:默认今天;用户说"补昨天的"就用昨天。目标文件 `${SPACE_ROOT}/YYYY/MM-DD.md`
2. 文件已存在 → 读出来,进入"追加模式":今天已经写过,问用户是想补充还是另起一段
3. 按 tone 风格开聊:从一个开放问题开始("今天过得怎么样?"),顺着用户的话走,**每轮最多一个追问**,聊 2-4 轮即可,不要审讯
4. 用户表达完后,把对话整理成第一人称日记草稿(保留用户的用词和口气,不要润色成公文),给用户过目
5. 用户确认后写入文件:frontmatter 的 mood/tags 只写用户明确表达过的;追加一行到 `${SPACE_ROOT}/.llm/index/entries.jsonl`:

```json
{"date":"2026-06-12","path":"2026/06-12.md","mood":3,"tags":["工作"],"summary":"<一句话>","words":420}
```

## 不做的事

- 不替用户虚构细节、不加没说过的感受
- 用户只想随手记一句时,别强行展开引导——记下来就好
