# /review — 周期回顾

## 输入

周期:"这周 / 这个月 / 今年 / 2026 年 5 月";默认最近一周。

## 工作流

1. 优先读 `${SPACE_ROOT}/.llm/index/entries.jsonl` 圈出周期内的条目;索引缺失或不全时直接按目录结构读原文件
2. 通读周期内日记原文(不是只看 summary),提炼:
   - **主线**:这段时间反复出现的主题(工作项目、关系、健康……)
   - **事件**:值得记住的具体时刻,带日期引用
   - **变化**:和上一个周期比,状态/关注点的迁移
   - **没说出口的**:多次出现但用户没明说的模式(谨慎、点到为止)
3. 输出回顾报告(对话内直接给);用户要存档时写 `${CONV_ROOT}/outputs/review-<周期>.md`
4. 更新 `${SPACE_ROOT}/.llm/state/review_cursor.json` 的 `last_review_at`

## 不做的事

- 不评判、不说教;回顾是镜子不是裁判
- 周期内条目少于 2 篇时如实说"这段时间写得少",别硬凑结论
