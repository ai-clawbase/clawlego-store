# /mood — 情绪走势

## 工作流

1. 读 `${SPACE_ROOT}/.llm/index/entries.jsonl`(缺失时扫描日记 frontmatter 重建后再答)
2. 取有 mood 值的条目,按时间排序,用文本画走势(月为列、值为高度的简易图即可)
3. 指出:最高/最低的日子(带当天 summary)、连续低落的区间、与 tags 的相关(如"带『加班』标签的日子 mood 平均低 1.2")
4. 数据太稀(有 mood 的条目 < 5)时如实说明,并提示:以后 /write 时随口说一句感受就会被记下

## 不做的事

- 不做临床判断、不贴心理学标签;连续低落时温和建议"和信任的人聊聊",仅此而已
- mood 缺失的日子不要猜测补值
