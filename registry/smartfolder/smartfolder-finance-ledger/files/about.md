一本随口就能记、却有传统记账软件一身本事的账:多账户、净值、预算、现金流、周期账单、报销、
储蓄目标、多币种。"昨天打车 38"一句话就是一笔账;拖一张账单截图或银行 CSV 进来就能批量入账。
而且——文件夹永远属于你,我们的一切都收在 `.llm/` 里。

# 你能用它做什么

- "昨天打车 38,午饭 25 走支付宝"(/record 随口记)
- 拖入支付宝/微信/银行账单截图或 CSV → 批量入账(/import)
- "我有招行卡、信用卡、支付宝"(/account 建账户,看总净值)
- "这个月花哪儿了?净值多少?"(/report 月报 + 现金流 + 净值)
- "餐饮预算 1500,帮我盯着"(/budget)
- 房租工资订阅(/recurring)· 报销跟踪(/reimburse)· 攒钱计划(/goal)· 查账(/search)

# 文件夹结构:根目录干净,机制全在 .llm/

```
<账本根>/
├── llm_output/                ← 唯一对你可见的产物(报表、导出)
└── .llm/
    ├── template/              ← 模板:数据契约 schemas/ + 升级 migrations/(随版本可整体替换)
    ├── data/                  ← 你的资产:records/ 账目 + accounts/budget/recurring/goals 等设置
    ├── index/ · cache/        ← 派生层:索引与解析缓存(可重建,不占真相)
    └── ui/                    ← 工作台视图(总览/报表/预算/账户)
```

# 三件让它"软件级"的事

- **账目只追加、永不改写**:记错用冲正行更正,历史永远可审计。
- **带版本、可升级**:模板有版本号,新版到来时 `/upgrade` 原地迁移,旧账目零丢失;
  账单图/CSV 的解析按内容哈希缓存,跨版本自然复用,向前兼容。
- **派生可重建**:索引、净值、聚合都从 records 实时算出,坏了删了都能重建,真相只有一份。
