# /report — 报表

## 输入

月份("这个月"/"2026-05",默认当月);"今年"出年度视图;也接受"现金流"/"净值"专题。

## 工作流

1. 优先读 `.llm/index/monthly.json`;若 `index.schema` 新鲜度校验不过(records_digest 不符或
   schema_version 不等)→ 先 `/index` 重建,再出报表。年报读全年各月条目。
2. 统计前从 `.llm/entities/transaction/records.jsonl` 读原始数据,冲正行(`void_of`)与原行对消;
   `status:void` 不计;转账不计入收支只动账户余额。
3. 输出:
   - **总收支与结余**;**净值**(各账户余额合计,跨币种按 `fx.base_amount` 折本位币)。
   - **分类占比**(文本条形图)+ 环比上月增减最大的两类。
   - **现金流**:本期各账户流入/流出与期末余额。
   - **异常大额**:超过该分类月均 3 倍的单笔,逐条列。
   - **预算对照**(budget.json 存在时):每类 已用/预算/剩余。
4. 同步把这些聚合写进 `.llm/ui/schema.json` 的对应页(总览/报表/预算/账户),刷新工作台视图。
5. 用户要存档 → 写 `${SPACE_ROOT}/llm_output/report-<月份>.md`(对用户可见、可带走)。

## 不做的事

- 数据不足(当月 < 5 笔)如实说,不硬出占比图。
- 不做"你花太多了"式说教;预算超了陈述事实即可。
