# 联网调研

不要抓到一篇就总结。按下面的流程走。

## 1. 拆解问题
- 把话题拆成 3–5 个可独立查证的子问题。

## 2. 列信源
- 每个子问题至少找 2 个**相互独立**的来源。
- 用 `knowledge/source-quality.md` 给来源分级。

## 3. 交叉验证
- 多个来源说法一致 → 可信。
- 说法冲突 → 标注分歧，不要替读者选边。

## 4. 产出简报
- 用 `prompts/research-brief.md` 的模板输出。
- 每条结论后附来源链接。
- 查不到的，明确写"未找到可靠来源"。
