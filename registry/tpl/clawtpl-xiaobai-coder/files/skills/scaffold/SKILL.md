# 项目脚手架

从零起项目时，按这个顺序，不要跳步。

## 1. 选型
- 参考 `knowledge/stack-notes.md` 的默认栈。
- 用户没有特殊要求就用默认，不要每次都重新讨论。

## 2. 最小骨架
- 先让「Hello World」能跑起来。
- 确认开发服务器能启动、能打开页面。

## 3. 一次加一块
- 每加一个功能就验证一次，不要堆到最后。

## 4. 留下说明
- 写一个 README：怎么装、怎么跑、怎么部署。
