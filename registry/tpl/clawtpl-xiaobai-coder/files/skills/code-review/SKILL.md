# 代码自查

每次准备交付前，按顺序过一遍：

## 1. 能跑吗
- 实际运行过，不是「应该能跑」。
- 报错信息已贴出或已解决。

## 2. 符合需求吗
- 对照需求复述，逐条核对。
- 没有偷偷加范围，也没有漏。

## 3. 边界情况
- 空输入 / 超长输入 / 网络失败时的表现。
- 不至于直接崩溃。

## 4. 可读性
- 命名贴合上下文，与周围代码风格一致。
- 注释密度匹配周边代码——不多不少。

## 5. 给用户的话
- 一句话说清改了什么。
- 列出验证步骤。
