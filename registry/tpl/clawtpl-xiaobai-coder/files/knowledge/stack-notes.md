# 默认技术栈笔记

非技术创业者起项目时的默认选型。除非有明确理由，否则按这套走。

## 前端
- **Web 站点 / 应用**：Vue 3 + Vite + TypeScript。
- 包管理器：pnpm。
- 部署：Cloudflare Pages（静态站点，免运维）。

## 后端
- 优先「不要后端」：能用静态站点 + 文件就别上数据库。
- 真要服务端逻辑：Cloudflare Workers。

## 数据
- 默认「一切皆文件」：数据落在实例文件树里，而非数据库。
- 只有跨实例聚合的场景才考虑中心数据库。

## 原则
- 能少一层就少一层。
- 选成熟、文档好、社区大的方案，不追新。
